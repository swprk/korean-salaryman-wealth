# Contributing Guide

Korean Salaryman Wealth 스킬에 기여해주셔서 감사합니다. 이 문서는 효과적으로 기여하는 방법을 안내합니다.

## 🎯 기여할 수 있는 영역

### 1. 잘못된 수치 정정
세제·금리·한도 등의 수치가 잘못된 경우. **공식 출처 링크 필수**.

### 2. 누락된 영역 보완
- 새로운 사용자 시나리오 (예: 자영업자, 외국인 거주자, 군인 등)
- 새로운 정책 제도 (정부 발표 직후)
- 빈출 사용자 질문에 대한 답변 패턴

### 3. 세제 변경 반영
- 매년 1월 세법 개정안 적용
- 정책 출시 시점 (청년형 ISA 출시 등)
- 한도 변경, 공제율 변경

### 4. 답변 품질 개선
- 사용자 답변 예시의 명확성 향상
- 정정 톤 사례 추가
- references 파일의 가독성·정확성

### 5. 도구 개선
- `scripts/refund_calculator.py` 기능 확장
- 새로운 시뮬레이션 추가

## 📋 기여 프로세스

### Issue 먼저 등록

큰 변경(새 references 파일 추가, SKILL.md 구조 변경 등)은 **먼저 Issue로 논의**해주세요. 작은 수정(오타, 명백한 수치 오류)은 바로 PR도 환영합니다.

### Issue 작성 가이드

**잘못된 수치 발견 시**:
```markdown
## 발견한 오류
- 파일: references/youth-programs.md (라인 X)
- 현재 내용: "월 납입 한도 600만원"
- 올바른 내용: "월 100만원, 연 1,200만원"

## 공식 출처
- 주택도시기금 공식 페이지: https://nhuf.molit.go.kr/...
- 관련 법령: 조세특례제한법 제87조

## 발견 경위 (선택)
가입 직전 공식 사이트 확인 중 발견
```

**새 영역 제안 시**:
```markdown
## 제안 영역
자영업자 노란우산공제 통합 안내

## 왜 필요한가
현재 스킬은 직장인 위주이지만, 자영업자도 절세계좌 활용에 관심 많음.
사용자 질문에 자영업자 대응이 부족함.

## 다룰 내용 (예상)
- 노란우산공제 한도 + 소득공제
- 자영업자 IRP 추가납입
- 종합소득자 4,500만원 분기점 적용
```

## 🔍 검증 우선 원칙

이 프로젝트의 가장 중요한 원칙입니다.

### 모든 수치는 공식 출처 링크 필수

✅ **인정되는 출처**:
- 국세청 (hometax.go.kr, nts.go.kr)
- 금융위원회 (fsc.go.kr)
- 금융감독원 (fss.or.kr, 통합연금포털 100lifeplan.fss.or.kr)
- 국토교통부 (molit.go.kr)
- 주택도시기금 (nhuf.molit.go.kr)
- 국민건강보험공단 (nhis.or.kr)
- 국민연금공단 (csa.nps.or.kr)
- 금융투자협회 ISA다모아 (isa.kofia.or.kr)
- 청약홈 (applyhome.co.kr)
- 정부 정책 발표 보도자료 (대한민국 정책브리핑 등)

⚠️ **참고만 가능한 출처** (수치 단정 근거로 부족):
- 증권사·은행 블로그
- 재테크 블로그·카페
- 뉴스 기사 (정부 발표 인용 시만 OK)

❌ **인정 불가**:
- 출처 없는 수치
- 개인 의견·추측
- 유튜브·소셜미디어 정보

### 출처 표기 형식

references 파일에 변경할 때:

```markdown
- **납입 한도**: 월 최대 100만원, 연 1,200만원
  - 출처: [주택도시기금 청년주택드림청약통장 상품안내](https://nhuf.molit.go.kr/FP/FP07/FP0701/FP07010301.jsp)
  - 적용 법령: 조세특례제한법 제87조
```

## 🧪 테스트

### 환급 계산기 변경 시

```bash
# 기존 테스트 케이스 통과 확인
python scripts/refund_calculator.py --salary 5000 --pension 600 --irp 300
# 예상: 환급 148.5만원

python scripts/refund_calculator.py --gain 1200
# 예상: 양도세 209만원
```

### 답변 품질 변경 시

PR 설명에 최소 1개의 답변 예시 시뮬레이션 포함:

```markdown
### 테스트 시나리오
"32세 직장인, 연봉 4,800만원, 결혼 5년 내 가능성, 절세계좌 0개"

### 변경 전 답변 (요약)
- 일반 ISA 권장 + 연금저축 부분 납입

### 변경 후 답변 (요약)
- 청년주택드림 1순위 추가 (만 34세 이하)
- 일반 ISA 서민형 자격 자동 안내
- 부부 각자 한도 활용 가능성 짚음
```

## 🎨 스타일 가이드

### 마크다운

- 문장 끝 마침표 일관성
- 표 사용 시 `|` 정렬
- 코드 블록 언어 명시 (```` ```bash ````, ```` ```python ````)

### 톤 (한국어)

- **존댓말 사용** (사용자 답변)
- **정중한 정정 톤** ("그건 흔한 오해예요" 권장, "그건 틀렸습니다" 회피)
- **구체적 숫자 우선** (추상적 표현 회피)

### 답변 구조 (SKILL.md 답변 예시)

1. 사용자 상황 진단 (1줄)
2. 권장 액션 단계별 (1단계, 2단계, ...)
3. "이번 주 할 일" 정리 (1줄)
4. 시점 표시 ("2026년 4월 기준이며 향후 변경 가능")

## 🚀 Pull Request 가이드

### PR 제목 형식

```
[영역] 변경 요약

예시:
[fix] 청년주택드림 한도 정정 (600만→1,200만)
[feat] 자영업자 노란우산공제 references 추가
[docs] README 가독성 개선
[refactor] tax-accounts.md 섹션 재배치
```

### PR 본문 템플릿

```markdown
## 변경 요약
한 줄 설명

## 관련 Issue
#XX (있다면)

## 변경 내용
- 파일 A: ...
- 파일 B: ...

## 공식 출처
- 출처 1: URL
- 출처 2: URL

## 테스트
- [ ] 기존 환급 계산기 작동 확인
- [ ] SKILL.md 줄 수 500 이하 유지
- [ ] CHANGELOG.md 업데이트
```

### 머지 기준

- ✅ 모든 수치에 공식 출처 링크
- ✅ CHANGELOG.md에 변경 내역 기록
- ✅ 기존 답변 패턴과 톤 일관성 유지
- ✅ Code review 1명 이상 승인
